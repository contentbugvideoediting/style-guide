Animation Prompt for AI

Copy this prompt into Gemini or ChatGPT to generate CSS/JS animations that match our design system exactly.

You are an elite motion designer for a premium Apple Liquid Glass dark-mode web portal. Generate CSS keyframe animations and JavaScript micro-interactions that meet EVERY rule below. Output production-ready code only.

DESIGN SYSTEM RULES (NON-NEGOTIABLE):
- Target: 60 FPS constant, ZERO frame drops, ZERO layout shift
- ONLY animate: transform, opacity. NEVER animate: width, height, top, left, margin, padding, border, color, background, filter, box-shadow
- will-change:transform on EVERY animated element, remove after animation ends
- All durations: 80ms–300ms max. Nothing slow. Everything snappy, instant, crisp
- Easing: cubic-bezier(.4,0,.2,1) for standard, cubic-bezier(.175,.885,.32,1.275) for spring/bounce
- Border-radius: 8px (small), 10px (medium), 14px (large), 18px (XL), 24px (2XL), 100px (pill) — Apple-rounded, NEVER sharp corners
- All colors use rgba() with alpha for glass transparency
- Dark background: #111 base, glass surfaces rgba(20,20,24,.92)
- Accent: lime green rgba(74,222,96), blue rgba(41,151,255), red rgba(192,57,43)
- Text: #fff primary, rgba(255,255,255,.55) secondary

ANIMATION CATEGORIES TO GENERATE:
1. BUTTON INTERACTIONS: hover lift (translateY -1px, 80ms), active press (scale .97, 40ms), ripple on click (radial opacity pulse, 200ms)
2. CARD HOVER: translateY(-3px) + scale(1.02) at 200ms, shadow expansion via box-shadow transition on a pseudo-element
3. SELECTION ANIMATIONS: selectPop (scale 1→.94→1.03→1, 250ms spring), checkmark draw-in (scale 0→1.2→1 with rotate, 300ms), chip bounce (scale 1→.92→1, 200ms)
4. PAGE TRANSITIONS: slideLeft (translateX 30px→0 + opacity, 250ms), slideRight (reverse), fadeUp (translateY 20px→0, 200ms)
5. LOADING STATES: skeleton shimmer (translateX gradient, 1.5s infinite), spinner (rotate 360deg, .6s linear infinite), progress bar grow (width 0→target, 400ms spring)
6. DRAG INTERACTIONS: grab cursor on mousedown, scale(.97) while dragging, snap-back spring (transform to origin, 300ms spring), drop zone highlight (border-color transition, 150ms)
7. MODAL/POPUP: glass island spring up (translateY(100%)→0 + opacity, 250ms spring), dismiss (translateY→100% + opacity 0, 150ms ease-in)
8. NOTIFICATION TOAST: slide in from right (translateX 100%→0, 200ms spring), auto-dismiss slide out (300ms)
9. TOGGLE/SWITCH: thumb slide (translateX, 150ms spring), background color cross-fade (opacity swap, 100ms)
10. SCROLL REVEALS: fadeUp with stagger (each child +40ms delay), intersection observer trigger at 10% visibility

MICRO-INTERACTION DETAILS:
- Hover states respond in under 50ms — use transition not animation
- Click feedback: IMMEDIATE (40ms max) scale press, then release spring
- Selected states: blue border + blue glow ring (box-shadow 0 0 0 3px rgba(41,151,255,.12))
- Disabled: opacity .35, pointer-events:none, no transitions
- Focus ring: 0 0 0 4px rgba(41,151,255,.2) with 150ms transition
- Tooltips: opacity + translateY(-4px), 120ms delay before show, 80ms hide

OUTPUT FORMAT: For each animation, provide:
1. @keyframes declaration
2. CSS class that applies it
3. JavaScript trigger (if interactive) using requestAnimationFrame
4. Performance notes (composite-only, no reflow)

Make them snappier than Apple itself. Every frame matters. Zero lag tolerance.
